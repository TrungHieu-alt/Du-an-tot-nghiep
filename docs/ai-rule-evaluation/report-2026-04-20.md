## Evidence Highlights

### H1 (Hallucination Control)
Strengths:
- Canonical rule priority and conflict handling are explicit (`AGENTS.md:5-12`).
- Mandatory startup + targeted loading reduces blind inference (`AGENTS.md:13-27`, `AGENTS.md:71-74`).
- Escalation and hard stops for ambiguity/security/API risk are clear (`AGENTS.md:76-87`).
- Constraints disallow inference of missing docs (`docs/agent-rules/doc-map.md:85-88`).

Gaps:
- No explicit requirement that major technical assertions in handoff must cite exact file/path evidence.
- No explicit policy for what to do when required evidence cannot be found besides general escalation language.

### C1 (Context Sufficiency)
Strengths:
- Deterministic read order and startup routine (`AGENTS.md:13-23`, `docs/agent-rules/doc-map.md:7-14`).
- Workflow ownership paths are well mapped (`docs/agent-rules/codemap.md:9-62`).
- Task router and loading walkthrough exist (`docs/agent-rules/doc-map.md:42-49`, `79-83`).
- Playbook gives task-type procedures (`docs/agent-rules/playbook.md:10-57`).

Observed risk (non-blocking):
- `doc-map` routes only to matching-centric HLD nodes; non-matching workflows rely mostly on codemap + source inspection.

### S1 (Skill/Process + DoD Reliability)
Strengths:
- DoD contains strict mandatory evidence gates (`docs/agent-rules/definition-of-done.md:12-21`).
- Failure condition explicitly marks incomplete tasks as not done (`docs/agent-rules/definition-of-done.md:38-44`).
- Working contract enforces reproducibility and decision logging (`docs/agent-rules/working-contract.md:5-16`, `31-34`).
- API contract discipline is explicit (`docs/agent-rules/working-contract.md:18-25`).

Gaps:
- No explicit skill matrix linking task classes to required competencies/tools.
- No explicit repeatability/calibration checks (same scenario, same verdict expectation).
